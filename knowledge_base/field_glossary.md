# 字段口径说明

## Circle 使用数据表 (circle_data.csv)

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | string | 用户唯一标识，格式为 U 开头 + 3 位数字，如 U001 |
| circle_name | string | 圈子名称，目前有：早起打卡、运动挑战、读书会、冥想空间、英语角 |
| date | string | 活动日期，格式 YYYY-MM-DD，数据范围通常为一周 |
| duration_min | int | 用户在该圈子的使用时长，单位为分钟，取值为 0-120 的整数 |

## 用户映射表 (user_mapping.xlsx)

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | string | 用户唯一标识，与主表关联键 |
| user_name | string | 用户显示名称，如 "用户_1" |
| department | string | 用户所属部门，目前有：产品部、研发部、运营部、设计部 |

## 数据质量注意事项

1. **脏数据规则**: duration_min < 5 的记录可能是用户切到后台产生的无效记录，分析时需酌情排除
2. **缺失日期**: 并非每个用户每天都有数据，这是正常的，不要当做数据缺失
3. **测试用户**: user_id 包含 "test" 字样的为测试账号，正式报表需排除
4. **编码**: CSV 文件使用 UTF-8-BOM 编码，中文列名可直接使用
